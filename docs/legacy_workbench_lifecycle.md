# Legacy Workbench 生命周期与迁移清单

## 决策

React 应用的 `/projects` 是 Paper Agent 唯一正式用户入口。静态页面
`app/apps/frontend/public/paper-writer-workbench.html` 定义为 **Legacy / Prototype**：它只用于验证尚未迁入正式界面的工作台交互，不再作为默认可发现入口，也不再承担正式产品规范。

后端默认不提供 `/writing-workbench` 和 `/paper-writer-workbench.html` 的原型内容。只有运维或开发者显式设置以下开关并重启后端后，两个兼容路径才会返回原型页面：

```dotenv
OPENPRISM_ENABLE_LEGACY_WORKBENCH=true
```

除精确的 `true`（大小写不敏感、允许首尾空格）外，其余值都按关闭处理。关闭时后端对两个路径显式返回 404，避免前端静态文件服务或 SPA fallback 意外公开原型；Vite 开发服务器和 preview 使用同一规则拦截路径，默认构建也不会把原型复制到 `dist/`。若需要在独立 Vite 构建中保留原型，必须在执行构建时设置同一开关。启用后的后端响应带有 `X-OpenPrism-Legacy: true`、`X-Robots-Tag: noindex, nofollow` 和 `Cache-Control: no-store`；页面顶部也必须持续显示 Legacy / Prototype 提示和正式 `/projects` 入口。

正式 React UI 不得链接该原型。需要临时验证原型时，应直接输入兼容路径，并在测试或问题记录中注明已启用 feature flag。

## 功能所有权

以下能力由正式 React UI 持有，不应回流到原型形成第二套实现：

- 项目列表、创建、重命名和项目导航；
- 文件树、编辑器、上传、预览和 LaTeX 编译；
- Chat / Agent / Tools 会话和 Skill 选择；
- Corpus 导入、索引、检索和文档状态；
- Draw、终端、设置和移动端交互。

原型仍包含下列尚未完全迁入正式 UI 的独有展示或编排能力。迁移应复用现有后端 API/服务，不复制静态页面里的状态机或 DOM 逻辑。

| 原型独有能力 | 正式 UI 目标 | 迁移完成的验收条件 | 状态 |
| --- | --- | --- | --- |
| `writing-workbench/context` 聚合结果：任务路由、上下文摘要、Skill 决策、写作计划 | AI 助手内的“写作准备”视图 | React 组件展示后端结构化字段；项目切换和输入变更会使旧结果失效；有组件测试与 Playwright 主路径 | 待迁移 |
| 论文流程、操作队列、模式操作中心、生产可用性 Gate | AI 助手顶部的步骤与阻塞提示 | 用户能看到当前步骤、阻塞原因和安全确认；不得自动发送、执行或写文件 | 待迁移 |
| RAG 修复向导、文档引用可用性、检索改写和 OCR/摘录恢复队列 | 正式 Corpus 面板 | 上传诊断、文本预检、OCR 队列和检索重试在 React UI 可完成；失败态有真实 API E2E | 待迁移 |
| 证据包、引用策略、验收清单和完整工作包复制 | 正式 AI 证据抽屉 | 所有复制内容来自同一后端上下文且带证据指纹；输入变化后旧内容不可继续复制 | 待迁移 |
| `review-answer` 修订计划、修订循环和进展对比 | AI 回复审查面板 | 审查结论、阻塞项、修订 prompt 和复审历史可见；编辑回复后旧审查失效 | 待迁移 |
| `claim-review` 单句证据检查 | Corpus/AI 联合证据检查器 | 能检查来源编号、弱匹配与超范围主张；结果始终禁止自动写入正文 | 待迁移 |
| `adoption-package` 安全采纳包 | 编辑器的人工采纳流程 | 只生成可审查 diff/手工清单；最终写入仍需用户明确确认 | 待迁移 |
| 演示数据和浏览器端 fallback | Storybook/fixture 或测试辅助 | 不进入生产包；正式 UI 的错误态由真实契约测试覆盖 | 计划淘汰 |

每迁移一行，必须同步更新本表、`docs/func_list.md` 和对应红绿灯测试。只在原型增加新产品能力不算完成迁移；新能力默认应直接进入 React UI。

## 弃用阶段

1. **隔离（当前阶段）**：默认关闭路由、页面显示 Legacy banner、正式 UI 无入口。原型允许修复阻断迁移的缺陷，但不新增正式产品功能。
2. **迁移**：按上表逐项在 React UI 落地。每项至少包含组件/契约测试和一条真实浏览器主路径；后端 API 可以继续作为共享能力。
3. **冻结**：所有“待迁移”项验收通过后，原型改为只读参考；除安全问题外不再修改，连续两个发布周期仅保留显式 feature flag 兼容。
4. **移除**：两个发布周期后，确认正式 UI 无链接、文档无依赖、测试不再以原型作为产品验收入口，才可单独提交删除原型页面和兼容路由的变更。删除前必须保留后端共享 API，并在发布说明中给出 `/projects` 替代路径。

## 禁止事项

- 不得把 `OPENPRISM_ENABLE_LEGACY_WORKBENCH` 默认改为开启。
- 不得在正式导航、项目列表或设置页增加原型入口。
- 不得因原型仍能运行就跳过 React UI 的组件测试和 Playwright 验收。
- 不得在没有完成上表迁移审计时直接删除原型或其共享后端 API。
