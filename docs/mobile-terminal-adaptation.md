# 手机端终端适配方案

状态：Phase 1 已实现
最后更新：2026-06-07

## 背景

Coding Kanban 当前以桌面浏览器为主要使用场景，核心交互依赖真实键盘、鼠标滚轮和多列布局。手机端主要问题不是简单的屏幕变小，而是缺少可靠的 `Ctrl`、`Alt`、`Esc`、方向键、鼠标滚轮等输入能力，导致 agent 对话中断、历史输出浏览、TUI 操作和长 prompt 输入都不顺畅。

手机端适配目标是：在不破坏桌面端终端体验的前提下，为 touch 设备提供一层明确的“移动终端控制面”。

## 适配原则

- 桌面端优先保持现有键盘和 xterm 行为，不把移动端手势逻辑全局化。
- 移动端手势只作用于当前聚焦终端区域，不影响文件浏览器、VS Code iframe 和页面普通滚动。
- 手机端按钮优先发送真实终端输入字符，不模拟浏览器 `KeyboardEvent`。
- agent 对话优先支持“消息输入框 + 发送”，不要强依赖手机软键盘直接聚焦 xterm。
- 控制操作要面向用户语义命名，例如“中断”而不是要求用户理解 `Ctrl+C`。
- 第一版尽量纯前端实现，避免重启后端和改动已有协议。

## 推荐移动端布局

手机端聚焦视图建议从桌面分栏改为单会话全屏：

```text
顶部：返回 / 会话名 / 状态 / 切换会话
中间：xterm 终端区域，支持上下滑动、双指缩放、长按菜单
底部：agent 输入框，支持多行 prompt
底部快捷键条：中断 / Esc / Tab / Enter / EOF / 方向键 / 更多
```

窄屏下不建议常驻显示右侧“其他会话”栏。会话切换可以改成顶部下拉、底部抽屉或左右滑动切换。

## 核心功能清单

当前已落地手机端终端控制页，默认入口为 `/?view=mobile`，兼容 `/mobile`、`/m` 和 `#/mobile`：单会话全屏终端、会话切换、底部快捷键条、快捷键说明弹层、多行输入框、终端区域防下拉刷新、单指滚动 xterm scrollback、双指调整终端字号和“底部”按钮。终端触控监听使用捕获阶段的非 passive `touchstart/touchmove`，触屏设备在桌面聚焦页也启用同一逻辑。后续长按菜单、横滑切换和 TUI 鼠标滚轮模式仍作为增强项。

### 1. 移动端快捷键条

在聚焦终端底部显示一组 touch 按钮，只在窄屏或 `pointer: coarse` 设备启用。

建议第一批按钮：

| 按钮 | 终端输入 | 用途 |
| --- | --- | --- |
| 中断 | `\x03` | 等价 `Ctrl+C`，停止当前 agent 回复或中断命令 |
| EOF | `\x04` | 等价 `Ctrl+D`，结束输入流 |
| 挂起 | `\x1a` | 等价 `Ctrl+Z` |
| Esc | `\x1b` | 退出 TUI 当前状态 |
| Tab | `\t` | 补全或切换焦点 |
| Enter | `\r` 或 `\n` | 提交当前输入 |
| 上 | `\x1b[A` | 方向键上 |
| 下 | `\x1b[B` | 方向键下 |
| 右 | `\x1b[C` | 方向键右 |
| 左 | `\x1b[D` | 方向键左 |

`中断`按钮建议做防误触，例如长按触发、二次确认或短时间撤销提示。agent 对话场景下按钮文案应使用“中断”或“停止回复”，而不是只显示 `Ctrl+C`。

### 2. Agent 消息输入框

在终端底部提供普通 `<textarea>` 输入框，让用户可以用手机输入法稳定输入长 prompt。

推荐行为：

- 输入框支持多行编辑。
- 点“发送”后把文本加换行发送到当前终端。
- 发送后清空输入框并保持页面在聚焦会话。
- 可以支持“粘贴并发送”和“仅粘贴到输入框”两种操作。
- 对 shell 会话可以默认收起输入框，对 `codex`、`copilot`、`claude` 等 agent 会话默认展开。

第一版可以直接复用当前终端 WebSocket 输入通道；也可以复用已有 `/api/agent-sessions/:id/stdin` 路由，不需要新增后端接口。

### 3. 上下滑动浏览终端历史

手机端单指上下滑动终端区域，应控制 xterm scrollback，而不是滚动整个页面。

推荐行为：

- 单指上滑/下滑调用 `term.scrollLines(...)`。
- 滑到历史输出时暂停自动跟随底部，避免新输出强行把用户拉回最新行。
- 显示“回到底部”悬浮按钮。
- 点“回到底部”调用 `term.scrollToBottom()`，恢复自动跟随最新输出。
- 滑动时阻止页面 overscroll，避免浏览器地址栏和页面滚动干扰终端。

第一版默认只滚动 xterm 历史缓冲区，不向 PTY 转发鼠标滚轮事件。

### 4. 鼠标滚轮等价操作

移动端“上下滑动相当于鼠标滚轮”需要区分两种模式：

- 历史滚动模式：滑动只控制 xterm scrollback，适合 agent 对话和 shell 输出浏览。
- TUI 鼠标模式：滑动被转成 mouse wheel escape sequence 发送给 PTY，适合 `vim`、`less`、`htop` 等程序。

建议第一版只做历史滚动模式。TUI 鼠标模式作为第二版高级功能，放在“更多”菜单里手动切换，避免错误控制序列污染普通 shell。

如后续支持 TUI 鼠标模式，需要评估是否根据 xterm mouse tracking 状态自动判断，或由用户显式切换。

### 5. 双指放大缩小

双指 pinch 应调整终端字号，而不是触发浏览器页面整体缩放。

推荐行为：

- 双指放大：增加 `term.options.fontSize`。
- 双指缩小：减小 `term.options.fontSize`。
- 字号范围建议限制在 `11px` 到 `24px`。
- 字号变化后调用 `fitAddon.fit()`。
- fit 后通过现有 resize 通道同步 PTY cols/rows。
- 显示短暂提示，例如“字体 16px”。
- 字号偏好保存到 `localStorage`，可按设备或会话记住。

实现时应只在终端区域捕获 pinch，不影响 VS Code iframe、文件浏览器和页面本身。

### 6. 横向滑动切换会话

手机端可以用左右滑动切换上一个/下一个 agent，但应避免误触。

建议触发条件：

- 横向位移大于 `60px`。
- 横向位移和纵向位移比例大于 `1.8`。
- 当前没有在选择终端文本。
- 当前没有打开 VS Code 或文件浏览器等内部可滚动面板。

横滑切换只是切换前端聚焦会话，不向 PTY 发送任何输入。

### 7. 长按菜单

长按终端区域可以打开 touch 操作菜单。

建议菜单项：

- 粘贴
- 复制选中
- 中断
- 发送 Esc
- 发送 EOF
- 回到底部
- 清屏
- 调整字号

如果 Clipboard API 不可用，“粘贴”可以退化为聚焦底部输入框，让用户用系统粘贴能力输入。

## 是否需要改后端

第一版可以不改后端。

当前终端输入已经由前端通过 WebSocket 发送到 PTY，移动端快捷键、输入框、滑动滚动、回到底部、双指缩放都可以在前端完成。终端 resize 也已有通道，双指缩放后只需要复用现有 resize 逻辑。

不需要后端改动的能力：

- 发送 `Ctrl+C`、`Esc`、方向键、Tab、Enter、EOF 等控制字符。
- agent 消息输入框发送文本。
- 单指滑动滚动 xterm scrollback。
- 回到底部。
- 双指缩放终端字体。
- 前端保存字号和手势偏好到 `localStorage`。
- 横滑切换前端聚焦会话。
- 长按菜单中的大多数操作。

可能需要后端改动的能力：

- 把移动端偏好跨设备同步到服务端。
- 记录移动端控制操作的审计或遥测。
- 做“强制停止 agent 进程”，而不是发送 `Ctrl+C`。
- 新增 agent-aware 的中断/重启语义。
- 自动识别 TUI mouse tracking 状态，并决定是否把滑动转成鼠标滚轮事件。

因此第一期建议保持纯前端，避免重启后端服务。

## 实现拆分建议

### Phase 1：基础可用

- 新增 `MobileTerminalToolbar`。
- 在聚焦终端底部显示“中断 / Esc / Tab / Enter / EOF / 方向键”。
- 通过当前终端 WebSocket 发送控制字符。
- 加 `@media (hover: none), (pointer: coarse), (max-width: 768px)` 控制显示。
- 补移动端快捷键单元测试或组件测试。

验收标准：

- 手机端可点“中断”停止当前 agent 输出。
- 手机端可发送 Esc、Tab、方向键。
- 桌面端不显示移动快捷键条。

### Phase 2：滚动和缩放

- 新增 `TerminalTouchController`。
- 支持单指上下滑动滚动 xterm scrollback。
- 支持“回到底部”按钮。
- 支持双指 pinch 调整字号。
- 字号变化后 fit terminal 并同步 resize。
- 保存字号偏好到 localStorage。

验收标准：

- 手机上可以自然滑动查看历史输出。
- 双指放大缩小后终端字符清晰且 cols/rows 正确。
- 滚动终端不带动整个页面滚动。

### Phase 3：Agent 输入框

- 新增 `MobileAgentComposer`。
- 支持多行输入、发送、粘贴、清空。
- 支持按会话类型默认展开或收起。
- 发送后保持当前会话聚焦。

验收标准：

- 手机端可以用普通输入框向 agent 发送长 prompt。
- 发送文本不会丢字符，不依赖 xterm helper textarea 获得手机键盘焦点。

### Phase 4：高级手势和菜单

- 支持长按菜单。
- 支持横滑切换会话。
- 可选支持 TUI 鼠标滚轮转发模式。
- 增加移动端手势设置入口。

验收标准：

- 长按菜单操作不影响终端普通滑动。
- 横滑切换不会误触发送 PTY 输入。
- TUI 鼠标模式默认关闭，可手动开启。

## 主要风险

- xterm 手机端键盘和浏览器输入法兼容性不稳定，需要避免只依赖 xterm 聚焦输入。
- 过度拦截 touch 事件可能破坏页面滚动、文件浏览器和 VS Code iframe 操作。
- 双指缩放后如果没有及时 fit 和 resize，会导致终端显示列数与 PTY 实际列数不一致。
- `Ctrl+C` 误触会中断 agent，需要有视觉反馈或防误触机制。
- TUI 鼠标滚轮转发如果判断错误，会把控制序列写入普通 shell。

## 推荐默认体验

- 手机端默认进入单会话聚焦视图。
- 底部始终可见“中断”和“输入”。
- 高级按键收纳到“更多”。
- 单指滑动默认浏览历史输出。
- 双指缩放默认调整终端字号。
- VS Code 和文件浏览器在手机端以全屏 tab 或抽屉方式打开，不做桌面左右分栏。
