# SPE Submission Package

This directory contains materials for submitting Paper Agent to
**Software: Practice and Experience** (SPE, Wiley, ISSN 0038-0644).

SPE publishes papers on practical software development, emphasizing real-world
experience, design decisions, and lessons learned. The journal expects papers
that report on the conception, design, implementation, and evaluation of
substantial software systems.

## Prepared Artifacts

- `spe-paper.md` — Manuscript draft in Markdown.
- `manuscript/main.tex` — LaTeX manuscript conforming to Wiley SPE style.
- `manuscript/references.bib` — Bibliography.
- `architecture-diagrams.md` — Mermaid source for system architecture figures.

## SPE Manuscript Structure

1. Abstract
2. Introduction
3. Related Work
4. System Design and Architecture
5. Implementation
6. Evaluation and Experience
7. Discussion and Lessons Learned
8. Limitations and Future Work
9. Conclusion
10. Data Availability
11. Acknowledgements
12. References

## Official Constraints

- SPE is published by Wiley (not Elsevier).
- Manuscripts should emphasize practical software development experience.
- Papers typically range from 15–25 pages.
- The LaTeX template follows standard `article` class conventions;
  Wiley provides an official class file for final submission.
