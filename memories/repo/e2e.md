- This file is the repository bug record. Append one concise entry here every time a bug is fixed.
- Playwright E2E must manage frontend (3000) and backend (4000) separately; checking only Vite can reuse a broken environment where `/api` proxies return 500.
- On Ubuntu hosts missing Playwright Chromium GUI libs, a rootless workaround is: `apt-get download` the needed `.deb` packages, extract them into `.playwright-libs`, and run Playwright with `LD_LIBRARY_PATH` pointing at that local prefix.
- Focus-reporting terminal mocks must put stdin into raw mode before asserting on `CSI I/O`; otherwise focus in/out escape sequences can be line-buffered and produce false-red E2E failures.
- Never strip DA/DSR/OSC/DCS replies from live stdin on the way to the PTY: xterm.js auto-answers these capability queries via `term.onData`, and TUIs such as Copilot CLI block on that handshake and silently drop keystrokes if the replies are filtered out. Only sanitize *replay* content (server-side `sanitizeReplayForTerminal`), never live input.
- In split view, treat focused `iframe` elements (for VS Code Web and similar embedded editors) as intentional external focus targets. Root cause: `TerminalView` only treated native form controls as "intentional external focus", so after the user clicked from the terminal back into a VS Code `iframe`, the terminal blur/refocus guard saw no valid external target and immediately stole focus back. Fix: include `HTMLIFrameElement` in the external-focus allowlist and protect the flow with a Playwright regression that clicks terminal -> iframe -> types in the editor.
- The agent dashboard E2E suite shares backend session state; run Playwright with a single worker for stability.
- For ssh -> tmux sessions, `node-pty.resize()` alone is not enough on macOS; explicitly sending `SIGWINCH` to the PTY ild is required so ssh forwards the new size to remote tmux clients.
- Non-interactive tmux thumbnails must never send resize frames to the backend; cache the last live terminal geometry and scale a local preview instead, or tmux clients will shrink and render incorrectly.
- In React StrictMode, closing a WebSocket during CONNECTING in effect cleanup can produce false "closed before the connection is established" warnings and fake disconnect banners; defer close until onopen when cleaning up dev-only mounts.
- Shell handling must not assume zsh exists; prefer `SHELL`, then fall back across `bash -> zsh -> sh` for Linux/macOS compatibility.
- tmux path handling must auto-detect Apple Silicon Homebrew, Intel Homebrew, and PATH-based installs; avoid hardcoding only `/opt/homebrew/bin/tmux` in app code, scripts, or E2E tests.
- File-browser create dialogs must not use the draft name string itself as the open/closed flag. Root cause: clearing the input to an empty string unmounted the dialog before the user could finish typing. Fix: keep explicit create-dialog state, disable submit while the trimmed name is empty, and reuse the existing `saveTextFile` upload path to create empty files.
- Copilot CLI can fail even when the terminal itself is healthy if focus drifts to a focus-view static region and the app tries to recover only on `keydown`. Root cause: `AgentFocusView` relied on capture-phase synthetic `KeyboardEvent` forwarding after focus had already left the terminal; without `preventDefault()` this could duplicate the first character (`ssecond`), and when buttons were treated as input surfaces the redirect path was skipped entirely. Fix: treat buttons/anchors as non-text-inputs, reclaim focus on `pointerdown` for focus-view static regions, and keep regression coverage in `tests/e2e/copilot-focus.spec.ts` + `tests/e2e/tmux-enhancements.spec.ts`.
- `pnpm -r test` would hang indefinitely even after all assertions passed; root cause was that three service-level idle cleanup timers (`VsCodeWebManager.globalServer.idleTimer` in `apps/server/src/services/vscode-web-manager.ts`, `SftpService` pooled `connection.idleTimer` in `apps/server/src/services/sftp-service.ts`, `AgentSessionRegistry.awaitingInputTimers` in `apps/server/src/services/agent-session-registry.ts`) held the Node `--test-isolation=process` worker event loop open. Fix: call `.unref()` immediately after `setTimeout` at each of the three sites, and add two `hasRef()` regression tests (see AGENTS.md "后台定时器必须 `.unref()`" section). Any new “idle cleanup” timer must be `.unref()`-ed and covered by a `hasRef()` assertion.
- Dev-server ports and the Vite dev-server's proxy target for the Fastify backend must be env-driven (`HOST`, `PORT`, `WEB_HOST`, `WEB_PORT`, `WEB_BACKEND_HOST`, `WEB_BACKEND_PORT`). Do not hard-code `localhost:4000` / `0.0.0.0:3000` into `apps/web/vite.config.ts` or `scripts/restart-dev.sh`. The repo-root `.env` (git-ignored) is the single source of truth; `.env.example` is the committed template, and `.gitignore` must keep the `!.env.example` negation so the template stays trackable.
- The `marks direct sessions awaiting_input after screen stays unchanged` test in `apps/server/src/services/agent-session-registry.test.ts` is time-sensitive (~3s timer). Under heavy parallel load it can flake on the first attempt and pass on retry; treat isolated failures as load-related, not as logic regressions, but if it flakes repeatedly, shorten `awaitingInputIdleMs` in the test’s override rather than raising the global default.
