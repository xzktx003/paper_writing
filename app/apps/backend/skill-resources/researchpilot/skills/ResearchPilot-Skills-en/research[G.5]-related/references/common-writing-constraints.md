# 写作阶段通用约束

> 本文件适用于 G.1–G.6 所有写作阶段。每个 skill 触发时，以下约束全部生效。

---

## 触发时必须执行的准备步骤

每次触发 G.1–G.6 任意 skill，在写任何内容之前，必须按顺序完成：

### 1. 读全文手稿

读取 `docs/manuscripts/paper.md`（或 `paper.tex` / `paper-zh.tex`）当前全文，理解已写内容的结构、论点和术语。确保修改后不与已写部分产生割裂。

### 2. 读参考文献

读取 References 章节（md）或 `docs/manuscripts/references.bib`（LaTeX）中已有的全部文献条目，了解已引用了哪些工作。

### 3. 读架构注释

读取手稿文件开头的 `=== 论文架构 ===` 注释块，明确当前章节应写什么内容、放哪些图表。

### 4. 读 idea_report 和代码

读取 `docs/idea_report.md` 全文（重点是当前章节对应的 Part），以及 `docs/dev_log.md` 中的实验结果。如需了解具体实现，读取对应的代码文件。

### 5. 读写作范例（若有）

若 `docs/manuscripts/examples/style-notes.md` 存在，读取范例分析，遵循范例的结构和风格（范例优先于 Master-cai 模板的结构建议；写作规范约束不受范例影响）。

### 6. 确认写作思路

基于以上材料，向用户展示本次写作的思路，等用户确认后再写：

```
准备写 {章节名}。

**已读取**：
- 手稿当前内容（{N} 个章节，已写：{已写章节列表}）
- 参考文献（{N} 条）
- 架构注释中本章节的规划
- idea_report Part {N}（{相关内容摘要}）
- dev_log 实验结果（{关键指标摘要}）

**本章节写作思路**：
{从架构注释和 idea_report 中提取的写作规划，详细说明每个段落写什么}

**图表**：{本章节包含的图表，从架构注释中提取}

**拟引用文献**：{本章节计划引用的论文，来自 idea_report 的 References 和已有文献}

**需要你确认的问题**：
1. {若有需要用户决策的问题，列出 1-3 个；若无则省略}

思路确认后开始写作。
```

## 版本管理

写之前先备份手稿：

```bash
cp paper.md paper_{mm-dd_hh-mm}.md
# 或
cp paper.tex paper_{mm-dd_hh-mm}.tex
```

写完后在文件头追加修改记录：

**md**：
```markdown
<!--
修改时间：{mm-dd_hh-mm}
修改内容：{写了/改了哪个章节，改了什么}
上一备份：paper_{上一次时间戳}.md
-->
```

**LaTeX**：
```latex
% 修改时间：{mm-dd_hh-mm}
% 修改内容：{写了/改了哪个章节，改了什么}
% 上一备份：paper_{上一次时间戳}.tex
```

## 批注扫描与处理

写之前扫描全文中所有批注：

**md**：扫描所有 `> %批注：` 开头的行
**LaTeX**：扫描所有 `%批注：` 开头的注释

处理规则：
- 按批注意见修改对应段落，修改后删除已处理的批注
- 批注有歧义（表述不清或存在多种理解）→ 先询问用户再修改
- 批注影响其他章节（如"这里的贡献和 Introduction 不一致"）→ 告知用户，建议同步处理

## 写作规范（全局强制）

1. **术语首次出现写全称**：首次出现时写"全称（缩写）"，后续直接用缩写。例："Transformer (TF)"。全文缩写定义不重复。

2. **每段一个信息**：每个段落只传达一个核心信息，段首句概括全段内容。

3. **数据必须真实**：所有定量数据（指标数值、数据集统计、超参数）必须从 `idea_report.md`、`dev_log.md` 或 `results/` 中取得，禁止捏造或估计。

4. **claim 必须有支撑**：论文中的每个主要论断都必须有实验数据或文献支撑。

5. **图表引用格式一致**：图用 "Fig.N"，表用 "Table N"，全文统一。

## 参考文献维护

写作过程中发现需要引用新文献时，立即追加到 References 章节（md）或 `.bib` 文件（LaTeX）。

**md 格式**：
```markdown
[N] {作者}. "{标题}." *{期刊/会议}*, {年份}.
> 核心工作：{这篇论文做了什么，一句话，说明其核心贡献}
> 引用原因：{在论文哪个位置引用，具体说明为什么在此处引用它}
```

**LaTeX .bib 格式**（每个 entry 上方）：
```bibtex
% [核心工作] {这篇论文做了什么，一句话}
% [引用原因] {章节位置}：{为什么在此处引用}
@article{key,
  author = {...},
  ...
}
```

**约束**：
- 每条文献必须有"核心工作"和"引用原因"两项注释，缺一不可
- 所有引用必须真实存在，来自 `docs/papers/` 或经 web_search 核验
- 不捏造文献
